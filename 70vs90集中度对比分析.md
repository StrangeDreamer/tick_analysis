# 70%集中度 vs 90%集中度 对比分析

## 📊 数据字段说明

### 90%集中度（当前使用）
- **含义**: 90%的筹码分布在多大的价格区间内
- **计算**: (90%成本上限 - 90%成本下限) / 平均成本
- **范围**: 包含了90%的持仓筹码

### 70%集中度
- **含义**: 70%的筹码分布在多大的价格区间内
- **计算**: (70%成本上限 - 70%成本下限) / 平均成本
- **范围**: 包含了70%的持仓筹码（更核心的部分）

## ⚖️ 优劣对比

### 使用 90%集中度 ⭐⭐⭐⭐⭐ 推荐

#### ✅ 优点

1. **覆盖面更广**
   - 包含90%的筹码，代表绝大部分持仓
   - 能看到完整的筹码分布情况
   - 不容易漏掉重要信息

2. **更稳定可靠**
   - 数据更全面，不易受极端值影响
   - 能捕捉到大资金的整体布局
   - 业界标准用法，经过广泛验证

3. **适合中长线**
   - 适合识别大资金的建仓成本
   - 能看到主力的完整持仓区间
   - 更适合趋势交易

4. **筛选更合理**
   - 不会过于严格
   - 保留更多候选股票
   - 平衡了集中与分散

#### ❌ 缺点

1. **可能包含散户**
   - 边缘10%可能是散户筹码
   - 对主力控盘的判断稍弱

2. **不够精确**
   - 区间较大，成本不够明确
   - 对短线交易参考价值稍弱

---

### 使用 70%集中度 ⭐⭐⭐

#### ✅ 优点

1. **更精准识别主力**
   - 排除边缘20%散户筹码
   - 更能反映主力核心成本
   - 集中度指标更敏感

2. **成本区间更清晰**
   - 区间更窄，成本更明确
   - 支撑/压力位更精确
   - 适合短线交易参考

3. **筛选更严格**
   - 只保留主力高度控盘的股票
   - 质量可能更高

#### ❌ 缺点

1. **覆盖不全**
   - 只包含70%筹码，遗漏30%
   - 可能错过重要的持仓信息
   - 大资金分批建仓会被遗漏

2. **过于严格**
   - 会排除很多优质股票
   - 可能导致候选池过小
   - 不适合中长线布局分析

3. **数据不稳定**
   - 更容易受短期波动影响
   - 对异常交易更敏感

## 🎯 最终推荐

### 当前策略分析

**你的策略**: 量价齐升 + 多层筛选 + Tick分析

**目标**: 捕捉中短期趋势机会

**最佳方案**: **90%集中度 < 0.12** ⭐⭐⭐⭐⭐

### 为什么是 0.12 而不是 0.15？

1. **0.15 稍宽松**
   - 可能保留筹码过于分散的股票
   - 主力控盘不够明显
   
2. **0.12 更合理**
   - 既保留足够候选池
   - 又确保适度集中
   - 适合量价齐升策略

3. **0.10 太严格**
   - 会排除很多优质股票
   - 与量价齐升≥2天的放宽不匹配

### 为什么不用 70%集中度？

1. **覆盖不全**
   - 量价齐升是看整体趋势
   - 需要完整的筹码分布
   - 70%太片面

2. **不匹配策略**
   - 你的策略是趋势跟踪
   - 不是主力博弈
   - 不需要如此精确

3. **数据可用性**
   - AKShare提供90%数据
   - 业界标准就是90%
   - 70%参考意义不大
